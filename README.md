# Calibrated Uncertainty in Blood Glucose Prediction for Type 1 Diabetes

A Transformer-based neural network that produces both 30-minute blood glucose forecasts **and** calibrated confidence intervals for individuals with Type 1 Diabetes (T1D), evaluated on the OhioT1DM 2020 dataset.

This repository accompanies the [blog post](LINK_TO_BLOG_POST) describing the project.

---

## TL;DR

- A 5-head quantile regression Transformer predicts the 5th, 25th, 50th, 75th, and 95th percentiles of glucose 30 minutes in the future
- Conformal calibration converts these into prediction intervals with formal coverage guarantees
- We identify a previously unmeasured failure mode: standard conformal calibration **undercovers hypoglycemia** by 24 percentage points
- A range-specific calibration extension partially addresses this and reveals that prediction quality (r = −0.88 with coverage) is the root cause

**Final test set performance:**
| Metric | Value |
|---|---|
| RMSE | 19.53 mg/dL |
| MAE | 13.95 mg/dL |
| Clarke Error Grid A+B | 100.0% |
| Hypoglycemic 90% coverage (range-specific) | 78.2% |

## Dataset

This project uses the [OhioT1DM 2020 dataset](http://smarthealth.cs.ohio.edu/OhioT1DM-dataset.html) — a publicly available dataset containing approximately 8 weeks of training data and 2 weeks of test data per patient for 6 individuals with Type 1 Diabetes. Each patient's data includes 5-minute continuous glucose monitor (CGM) readings, insulin events, meal logs, and wearable sensor signals.

**Access:** The dataset is not redistributed in this repository due to its data use agreement. To obtain access, complete the request form on the [official dataset page](http://smarthealth.cs.ohio.edu/OhioT1DM-dataset.html). Once you have the XML files, place them in `data/OhioT1DM/2020/train/` and `data/OhioT1DM/2020/test/` to reproduce the results.

## Reproducing the results

The pipeline runs in three sequential steps:

```bash
# 1. Train the model (saves to checkpoints/best_model.pt)
python run_training.py

# 2. Run conformal calibration + evaluation on the test set
python run_evaluation.py

# 3. Generate all blog figures
python run_visualizations.py
```

Each script prints progress and final results to stdout. The training script uses early stopping and typically converges in ~25 epochs (~30 minutes on CPU).

## How the code is organized

The pipeline is structured as a series of small, focused modules. Each module has a single responsibility and can be tested independently.

### `tokenizer.py`

Converts raw OhioT1DM XML files into fixed-length token sequences. Handles the regular 5-minute grid construction, three-tier gap handling (interpolation for short gaps, masking for medium gaps, discard for long gaps), feature computation (glucose, rate of change, time-of-day encoding, carbs, interpolation flag), and sliding window extraction. Per-patient normalization statistics are computed from training data only.

### `dataset.py`

Wraps the tokenizer output in PyTorch `Dataset` and `DataLoader` objects. Maintains the chronological 70/30 train/validation split within each patient's training XML and keeps the official test XML completely held out. Patient identity is preserved through every sample for per-patient evaluation downstream.

### `model.py`

The Quantile Regression Transformer architecture. Approximately 111,000 parameters across a linear input projection, sinusoidal positional encoding, three Transformer encoder layers with 4-head self-attention, attention pooling, and five independent quantile heads.

### `loss.py`

The pinball (quantile) loss function. Asymmetric penalty causes each output head to converge to its target percentile of the conditional distribution. Also provides per-quantile loss tracking and empirical coverage computation for training-time monitoring.

### `trainer.py`

The training loop. Includes AdamW optimization, ReduceLROnPlateau scheduling, gradient clipping, early stopping (patience = 15 epochs), and checkpoint saving. Records full training history for post-hoc analysis.

### `calibrator.py`

Conformalized quantile regression. Implements both standard (marginal) conformal calibration and the range-specific extension that computes separate adjustments for hypoglycemic, normal, and hyperglycemic ranges. Provides methods to apply calibration to new predictions at inference time.

### `evaluator.py`

Test set evaluation. Computes point prediction metrics (RMSE, MAE, MBE), coverage metrics at 50% and 90% confidence levels (both global and range-specific), range-specific breakdowns (hypo, normal, hyper), and the Clarke Error Grid zone distribution. Returns a pandas DataFrame with per-patient and aggregate results.

### `visualize.py`

All visualization code. Each figure is generated by a self-contained function with clear inputs (typically a results DataFrame or saved model outputs) and a deterministic output path in `blog_figures/`.

## Methodology summary

For complete details see the [blog post](LINK_TO_BLOG_POST). At a high level:

1. **Tokenization** — 2-hour windows of CGM history are converted into sequences of 24 timesteps with 6 features each
2. **Model** — A Transformer encoder maps each window into 5 quantile predictions
3. **Training** — The pinball loss pushes each quantile head to learn its target percentile
4. **Calibration** — Conformalized quantile regression provides formal coverage guarantees
5. **Range-specific extension** — Separate conformal adjustments per glycemic range address the hypoglycemic miscoverage problem

---

## Limitations

Honest limitations that the results should be interpreted in light of:

- **Small cohort:** Only 6 patients in OhioT1DM 2020. Statistical power for inter-patient analyses is limited.
- **Missing insulin timing:** The 2020 release does not include bolus timestamps, so insulin dynamics are not modeled.
- **Distribution shift:** Range-specific calibration achieves target coverage on the calibration set but only partially on the held-out test set, reflecting drift between calibration and test periods.
- **Marginal guarantee:** Conformal prediction guarantees coverage marginally; conditional coverage within glycemic ranges holds under the exchangeability assumption applied to each subgroup.

## Future work directions

The results motivate three concrete next steps:

1. **Patient-specific fine-tuning** of the quantile heads on individual patient data, particularly targeting hypoglycemic episodes for patients whose hypo RMSE remains high
2. **Rolling conformal recalibration** during deployment to address temporal distribution shift
3. **Extension to the 2018 OhioT1DM release** which includes bolus timing data and 12 additional patients
